Technical Notes

Every pricing algorithm in VolLab is implemented from scratch. This page documents the math, the implementation decisions, and the limitations of each module.

From Scratch vs Library

Component Implementation Why
Normal CDF From scratch (math.erf) Avoids scipy.stats dependency for core pricing
Black-Scholes pricing From scratch The foundation — must understand every step
Analytical Greeks From scratch Closed-form partial derivatives of BS
Newton-Raphson IV solver From scratch Uses vega for quadratic convergence
SVI parameterization From scratch Formula, derivatives, arbitrage checks all hand-coded
Heston char. function From scratch Corrected formulation (Albrecher 2006)
Finite difference Greeks From scratch Central differences, model-agnostic
SVI fitting (optimization) scipy.optimize.minimize L-BFGS-B is a generic optimizer — reimplementing it adds no insight
Heston integration scipy.integrate.quad Adaptive quadrature is a solved problem
Calibration (DE + L-BFGS-B) scipy.optimize Global + local optimization — the objective function is custom

Module 1: Black-Scholes

d1 = [ln(S/K) + (r + σ²/2)·T] / (σ·√T)

d2 = d1 - σ·√T

C = S·N(d1) - K·e^(-rT)·N(d2)

P = K·e^(-rT)·N(-d2) - S·N(-d1)

N(x) = 0.5·(1 + erf(x/√2))

The normal CDF is computed via the error function rather than importing scipy.stats.norm. Edge cases handled: T=0 returns intrinsic value, σ=0 returns discounted intrinsic, deep ITM/OTM are clamped to avoid numerical overflow.

Limitations: Assumes constant volatility (the whole point of the subsequent modules is to relax this assumption), European exercise only, no dividends.

Reference: Black & Scholes (1973), Hull Ch. 15

Module 2: Implied Volatility (Newton-Raphson)

σ_{n+1} = σ_n - (BS(σ_n) - P_market) / vega(σ_n)

Initial guess: Brenner-Subrahmanyam — σ_0 ≈ √(2π/T) · P/S

Newton-Raphson is chosen over Brent's method because vega (the analytical derivative) is already available from Module 1, giving quadratic convergence for free. ATM options typically converge in 3-5 iterations.

For deep OTM options, the Brenner-Subrahmanyam guess can be too low, leading to near-zero vega and solver stagnation. The implementation uses multiple starting guesses (0.50, 0.20, 1.00 as fallbacks) to handle this robustly.

Limitations: Can fail for options priced below intrinsic value (arbitrage condition — returns NaN rather than a nonsensical IV).

Reference: Brenner & Subrahmanyam (1988)

Module 3: SVI Volatility Surface

w(k) = a + b·(ρ·(k - m) + √((k - m)² + σ²))

k = log(K/F), w = total implied variance = σ_iv² · T

5 params per slice: a (level), b (wings), ρ (skew), m (shift), σ (curvature)

Gatheral's SVI is chosen over SABR because it's the industry standard for equity index options and has analytically tractable no-arbitrage conditions. Fitting is non-convex, so 10 random starting points are used with the best result selected.

Butterfly arbitrage is checked via the implied density condition (Gatheral & Jacquier 2014). Calendar arbitrage is checked by verifying total variance is non-decreasing in T. Violations are displayed explicitly rather than hidden — showing awareness of the problem is more valuable than pretending it doesn't exist.

Limitations: SVI is fit per-slice independently, then calendar arbitrage is checked post-hoc. A joint surface fit (SSVI) would enforce calendar constraints during fitting but adds significant complexity. The raw least-squares objective can produce surfaces with butterfly violations in the wings — constrained optimization or Sobol starting points would improve robustness.

Reference: Gatheral (2004), Gatheral & Jacquier (2014)

Module 4: Heston Stochastic Volatility

dS = r·S·dt + √v·S·dW₁

dv = κ·(θ - v)·dt + ξ·√v·dW₂

corr(dW₁, dW₂) = ρ

Pricing: C = S·P₁ - K·e^(-rT)·P₂ via Fourier inversion of the char. function

The implementation uses the corrected formulation from Albrecher, Mayer & Schoutens (2006) — the "little Heston trap." The original Heston (1993) formulation has a branch cut discontinuity in the complex logarithm that causes numerical instability for certain parameter regimes. The corrected version rotates the complex square root to eliminate this.

The Feller condition (2κθ > ξ²) ensures the variance process stays strictly positive. When violated, variance can touch zero — this is common in real calibrations and doesn't invalidate prices, but would require careful treatment in Monte Carlo simulation (e.g., full truncation or QE scheme).

Limitations: Single-factor stochastic vol — can't capture the steep short-dated smile observed in markets. Extensions: add jumps (Bates model), rough volatility (fractional kernel), or local-stochastic vol (LSV) hybrid.

Reference: Heston (1993), Albrecher et al. (2006)

Module 5: Model Calibration

min_{v0,κ,θ,ξ,ρ} Σᵢ (σ_heston(Kᵢ,Tᵢ; params) - σ_market(Kᵢ,Tᵢ))²

Stage 1: Differential evolution (global search)

Stage 2: L-BFGS-B (local refinement)

Calibration minimizes the sum of squared IV errors between Heston-implied vols and market-observed vols. The objective is computed in IV space (not price space) because IV errors are more interpretable and give equal weight across strikes.

Differential evolution is used for the global search because the objective is non-convex with many local minima. L-BFGS-B then refines the best result. A soft penalty encourages the Feller condition without hard-constraining it (many real calibrations violate Feller).

Limitations: The IV-space objective requires inverting each Heston price through BS (expensive). A price-space objective with vega weighting would be faster. The calibrated parameters are point estimates with no uncertainty quantification — bootstrap or MCMC would give confidence intervals.

Reference: Gatheral "The Volatility Surface" Ch. 3

Module 6: Greeks — Analytical vs Finite Differences

Central difference: df/dx ≈ (f(x+h) - f(x-h)) / (2h) — O(h²) truncation error

Second derivative: d²f/dx² ≈ (f(x+h) - 2f(x) + f(x-h)) / h²

Optimal h ≈ √ε · S where ε ≈ 2.2 × 10⁻¹⁶ (machine epsilon)

BS has closed-form Greeks. Heston doesn't — finite differences are the only practical option (automatic differentiation is an alternative but adds complexity). The error analysis chart shows the V-shaped tradeoff: too large h gives O(h²) truncation error, too small h gives floating-point cancellation error from subtracting nearly equal numbers.

Comparing BS analytical vs BS finite differences validates the numerical method. The same validated method applied to Heston then gives Greeks that can't be obtained analytically.

Limitations: Central differences require 2 function evaluations per Greek (4 for Gamma). For Heston, each evaluation involves numerical integration, making this expensive. Adjoint algorithmic differentiation (AAD) would be the production approach for computing Greeks on a large book.

Reference: Hull Ch. 19, Glasserman "Monte Carlo Methods" Ch. 7

What I'd Do Differently

SVI fitting is sensitive to initial conditions — I'd explore quasi-random Sobol sequences for starting points instead of pseudo-random, and consider SSVI (Surface SVI) for joint cross-expiry fitting with built-in calendar arbitrage constraints.

The calibration objective works in IV space, requiring a BS inversion for every Heston price. A price-space objective with vega weighting would halve the computation cost.

The Heston characteristic function integration uses scipy.integrate.quad (adaptive Gauss-Kronrod). Gauss-Laguerre quadrature with ~32 nodes would be significantly faster for production use since the integrand decays exponentially.

For Greeks, AAD (adjoint algorithmic differentiation) via a library like JAX would compute all Greeks in one backward pass rather than 2N forward evaluations for N Greeks.

Extensions I'd Explore

Rough volatility: Replace the Brownian motion driving variance with a fractional Brownian motion (H < 0.5). The rough Bergomi model better captures the short-dated smile steepness that Heston misses.

Jump-diffusion (Bates): Add a Poisson jump process to the spot dynamics. Captures sudden large moves that diffusion models underestimate.

Local-stochastic vol (LSV): Combine a local volatility function with Heston stochastic vol. The industry standard for exotic pricing — perfectly calibrates to vanillas while maintaining realistic dynamics.

Monte Carlo with variance reduction: Antithetic variates, control variates (BS as control), and quasi-Monte Carlo (Sobol sequences) for pricing path-dependent exotics under calibrated Heston.
